备忘录模式也称为快照模式
1.意图：在不破坏封转性的前提下获取一对象的内部状态，并且在该对象之外保存该状态，在需要的时候将该对象还原到原先状态。
2.描述：备忘录模式一般会有3中角色：1)备忘录角色用来保存备忘录发起人的内部状态，备忘录发起人根据需要决定备忘录角色存储备忘录
                                                                      发起人的角色的哪些状态。备忘录角色会提供2个接口：宽接口和窄接口。宽接口是提供给备忘录发起人访问，能够访问到内部状态
                                                                      窄接口提供给备忘录管理者，该接口中备忘录存储的内部状态是不可见的。
                                 2)备忘录发起人角色用来产生备忘录对象并记录自己当前的内部状态以及在需要的时候恢复自己的内部状态。
                                 3)备忘录管理角色用来保存备忘录对象。
