桥接模式
1.意图：如果一个对象在两个或者两个以上的维度上进行变化，可以将这2个维度抽象出来让一个维护去组合另外一个维度。
2.好处：将2个维度进行解耦，当一个维度发生变化的时候不会影响到另外一个维度，同时能极大的减少类的数量。
3.场景：比如你开了家餐厅，餐厅提供牛肉面，羊肉面，猪肉面同时也提供各种口味：酸的，甜的，苦的，辣的。如果面向对象的继承技术，将会导致类的数据比较多，
               差不多等于2个维度子类数量的乘机。这个时候你就可以把它们分为2个维度，   一个维度处理各种面的变化，一个维度处理各种口味的变化，这个时候类的数量
               只是2个维度子类数量的和。
